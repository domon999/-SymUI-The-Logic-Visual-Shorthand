# 🚀 SymUI: The Logic-Visual Shorthand

### **为人类直觉设计，为 AI 解析优化。**

**「 极简 · 直观 · 高熵 · 秒懂 」**

---

### 为什么选择 SymUI？

* **👁️ 直观性 (Intuitive Design):** 符号即形状。`|_ _|` 是底部弹窗，`< >` 是点击热区。无需“脑补”，看到符号即看到布局。
* **🤖 AI 友好 (AI-Native):** 消除自然语言的歧义。为大模型提供 0 噪声的结构化输入，解析准确率提升 300%。
* **⚡ 效率革命 (Edit Efficiency):** 放弃笨重的原型工具。在记事本里敲出 1 行符号，等同于在设计软件里画 10 分钟。

## 核心符号表 (The Grammar)

## UI 基础符号：

<按钮>  /可点击的对象/ 

"文字"  /显示的纯文本/ 

[容器]  /容器/ 

(图标)  /图片或图标/ 

{悬浮}  /弹窗和悬浮容器/

&变量名  /动态数据变量。例如："欢迎，& 用户名 &"/ 

/备注/  /备注/ 

┏底部弹窗┓  /底部弹窗/

| /横向分割，当|出现在元素内部时，对容器内部进行横向分割/

|元素|元素|  /横向分割，当|出现在元素外部时，表示对行进行分割/

/  /竖向分割，可分割下拉选项和小容器内部的上下分割。当“/”出现在外部时，只表示备注信息/



## 组合符号：

<"文字">  /可点击的纯文字链接/

<(可点击的图片/图标)>  /可点击的图片或图标/

<["可点击的"]>  /可点击的大尺寸按钮 或 可点击的整行/

  <[(图片)"图片名"||<(菜单按钮)>]>表示该按钮整行都可以被点击，里面显示有左侧两个元素(图片)和"图片名"，右侧一个元素<(菜单图标)>

"["提示词"]"  /输入框。内部显示"灰色提示词"/

["状态1""状态2""状态3"]  /显示状态文字。不可点击，系统根据实际情况显示且只显示其中一种状态/

[→<标签1>|<标签2>|<标签3>]  /分段控制器，箭头指向→默认显示的选项/

<→["选项1"^/"选项2"/"选项3"/"选项4"]>  /下拉选项。逻辑：^ 后面的内容默认隐藏，点击后展开。

 [8:|2:]  /标记宽度比例/  /举例：[8:"["搜索商品"]" | 2:<"搜索">]/  /表示带元素宽度比例的单行搜索框，输入框占8份，搜索按钮占2份，输入框中默认灰色提示词“搜索商品”/

## 【 整行布局 】

  |<按钮>|表示按钮较小，并居中。

  <按钮>|表示按钮在左。

  |<按钮>表示按钮在右。

  <按钮><按钮>|表示两个按钮都在左。

  |<按钮>|<按钮>表示两个按钮分别在中和右。

  <按钮>|<按钮>便是两个按钮分别在左和右。

  <按钮>|<按钮>|<按钮>表示三个按钮分别在左中右。

  <按钮>|<按钮>|<按钮>|<按钮>表示四个按钮分别在左中中右。

  (<按钮>)*2|<按钮>表示大按钮占左和中占2/3宽度，小按钮在右占1/3宽度。

  (图片)|(图片)表示两张图片各占1/2宽度。

  (<按钮>)|(<按钮>)表示两个按钮各占1/2宽度。



## 跳转：

<按钮1> >> 【页面1】 /点击<按钮1>跳转至【页面1】 /

## 【 进度标签 】

   &1需求  &2文字  &3图形  &4开发  &5完成

  功能说明：这些进度标签的作用是让开发者能通过文档快速的了解项目的各个板块的进度。这些标签的用法是，将其中一个进度标签写到某个【页面】&1需求 和 组件<按钮>&4开发 的后面。



## 【 页面编号 】

  1.给所有页面前方写上编号，1.1【页面】、1.2.1【页面】、1.2┏形象视频详情┓，表示层级关系。2.能够快速找到跳转指定的页面。3.并且平铺列表的方式无需层层打开，只需打开一层即可，非常便于查找。
