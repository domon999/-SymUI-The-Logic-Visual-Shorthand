### 为什么选择 SymUI？

* **👁️ 直观性 (Intuitive Design):** 符号即形状。`|_ _|` 是底部弹窗，`< >` 是点击热区。无需“脑补”，看到符号即看到布局。
* **🤖 AI 友好 (AI-Native):** 消除自然语言的歧义。为大模型提供 0 噪声的结构化输入，解析准确率提升 300%。
* **⚡ 效率革命 (Edit Efficiency):** 放弃笨重的原型工具。在记事本里敲出 1 行符号，等同于在设计软件里画 10 分钟。

## 核心符号表 (The Grammar)

### 1. 容器：形状即语义

* **`[ ]` (Box):** 基础容器/输入框。
* **`{ | | }` (Modal):** 居中弹窗。封闭结构暗示全屏遮罩。
* **`|_ _|` (Tray):** 底部弹窗。开口向上，暗示从底部升起的托盘。
* **`*( )*` (Pop):** 气泡弹窗。`*` 代表触发点，`()` 代表轻量悬浮。

### 2. 交互：动作即逻辑

* **`< >` (Active):** 热区。被包裹的内容即刻产生“可点击”属性。
* **`>>` (Link):** 跳转。能量流动方向，指向目的地。
* **`^` (Expand):** 展开。下拉菜单的灵魂，表示内容的向下延展。

### 3. 布局：栅格即对齐

* **`|` (Pipe):** 空间分割符。
* **`[数字:数字]`:** 强制比例。例如 `[3:7]` 自动触发栅格对齐，AI 自动推导 Flex 权重。

---

## 状态极简协议 (Suffix States)

无需书写 `status: disabled`，只需一个后缀：

* **`~`** : 禁用 (Disabled) -> `<"提交">~`
* **`+`** : 选中 (Active) -> `<"男">+`
* **`!`** : 错误 (Error) -> `["手机号"]!`
* **`%`** : 加载 (Loading) -> `<"支付">%`
