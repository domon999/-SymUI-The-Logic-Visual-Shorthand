### **为人类直觉设计，为 AI 解析优化。**

**「 极简 · 直观 · 高熵 · 秒懂 」**

---

### 为什么选择 SymUI？

* **👁️ 直观性 (Intuitive Design):** 符号即形状。`|_ _|` 是底部弹窗，`< >` 是点击热区。无需“脑补”，看到符号即看到布局。
* **🤖 AI 友好 (AI-Native):** 消除自然语言的歧义。为大模型提供 0 噪声的结构化输入，解析准确率提升 300%。
* **⚡ 效率革命 (Edit Efficiency):** 放弃笨重的原型工具。在记事本里敲出 1 行符号，等同于在设计软件里画 10 分钟。

## 核心符号表 (The Grammar)

## 🎨 UI 基础符号 (Basic Elements)

| 符号 | 定义 | 描述 |
| --- | --- | --- |
| `<按钮>` | 点击对象 | 所有可交互的触发器 |
| `"文字"` | 纯文本 | 界面上显示的静态文本 |
| `[容器]` | 容器 | 组合多个元素的逻辑外框 |
| `(图标)` | 媒体 | 图片、矢量图标或 Avatar |
| `{悬浮}` | 弹出层 | 气泡、悬浮容器、临时窗口 |
| `&变量名` | 动态数据 | `"欢迎，&用户名"` -> 动态渲染占位符 |
| `/备注/` | 说明 | 对当前行或元素的补充解释 |
| `┏弹窗┓` | 底部弹窗 | 特定指代从底部升起的交互层 |

## 🧩 组合符号逻辑 (Composite Elements)

通过嵌套基础符号，描述复杂的组件：

* **文本链接：** `<"文字">` (可点击的纯文字)
* **图片按钮：** `<(图标)>` (可点击的图标)
* **整行按钮：** `<["文字"]>` (具有容器背景的大尺寸按钮)
* **复合容器：** `<[(图片)"标题" \|\| <(菜单)>]>`
* *逻辑：左侧包含图片和标题，右侧对齐菜单按钮。*


* **输入框：** `"["提示词"]"` (内部显示灰色 Placeholder)
* **状态标签：** `["待支付" "已支付" "已取消"]` (仅显示当前实际状态)
* **分段器：** `[→<标签1> \| <标签2>]` (箭头 `→` 指向当前激活项)
* **下拉菜单：** `<→["选项1" ^ / "选项2" / "选项3"]>`
* *逻辑：`^` 符号后的内容默认隐藏，点击后展开。*

## 📐 布局与比例 (Layout & Grid)

使用 `|` 进行空间分割，使用 `n:m` 定义宽度比例。

* **比例分割：** `[8:"["搜索"]" | 2:<"搜索">]`
* *8 份宽度分配给搜索框，2 份宽度分配给按钮。*


* **横向分割 (元素内)：** `|元素|元素|` (用于行内元素的并列)
* **竖向分割 (下拉/小容器)：** `/` (用于纵向排列的选项)

### 整行布局示意

* `<按钮>|` ：居左
* `|<按钮>|` ：居中
* `|<按钮>` ：居右
* `<按钮>|<按钮>` ：两端对齐
* `(<按钮>)*2|<按钮>` ：左侧占 2/3，右侧占 1/3

## 📱 页面与层级结构 (Containers & Navigation)

### 容器定义

* **【页面】**：独立页面级容器，下方缩进内容为其内部成员。
* **{居中弹窗}**：模态对话框。
* **{[大容器]}**：如用户列表、卡片组等列表型容器。

### 交互跳转 (Flow)

使用 `>>` 符号描述交互逻辑：

* `<返回按钮> >> 【首页】` (点击跳转页面)
* `<设置按钮> >> {气泡弹窗}` (点击弹出气泡)
* `<删除按钮> >> ┏底部确认弹窗┓` (点击弹出底部菜单)

## 🚀 项目管理与进度控制

### 进度状态标签 (Status Tags)

将标签写在【页面】或 <组件> 后方，实时标注开发状态：

* `&1 需求` | `&2 文字` | `&3 图形` | `&4 开发` | `&5 完成`

> **示例：** `1.2 【个人中心】 &4开发`

### 页面编号规范

1. **层级化：** 使用 `1.1`、`1.2.1` 标注页面所属关系。
2. **平铺性：** 文档采用平铺列表，无需多层点击，便于快速定位跳转目标。
3. **唯一性：** 每个编号对应唯一的页面或弹窗。
